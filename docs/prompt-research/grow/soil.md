# Grow - 土壌プロンプトリサーチ

## 概要

ユーザーが自分の栽培場所の土壌について調べるためのプロンプト。

## プロンプト一覧

### 1. soil_basic - 地域の土壌を調べる

**目的**: 地名から一般的な土壌タイプを特定する

#### 現行プロンプト（v1.0）

```
{location}で家庭菜園・農業をしています。

この地域で一般的な土壌の種類を教えてください。
以下の形式で回答してください：

1. 土壌の種類（日本語名）
2. WRB国際土壌分類での名称（英語）
3. 土壌の特徴（色、質感、水はけなど）
4. この土壌に適した野菜・作物
5. この土壌で注意すべき点
```

#### 設計意図

- **番号付きリスト指定**: 回答の構造を統一し、ユーザーが情報を見つけやすくする
- **WRB分類を明示的に要求**: Growアプリの土壌選択肢と一致させるため
- **実用的な情報を含める**: 適した作物、注意点で実際の栽培に役立てる

#### 改善検討事項

- [ ] 「家庭菜園・農業」の表現は曖昧か？
- [ ] 市区町村レベルで十分か、より詳細な住所が必要か？
- [ ] 日本以外の地域でも機能するか？

### 2. soil_detailed - 土壌を詳しく分析

**目的**: 観察結果から土壌を推定する

#### 現行プロンプト（v1.0）

```
私の畑の土壌について教えてください。

【土壌の観察結果】
- 場所：{location}
- 色：{color}
- 手触り：{texture}
- 水はけ：{drainage}
- その他の特徴：{other}

この情報から：
1. 推定される土壌タイプ（WRB分類）
2. 土壌のpH傾向
3. 適している農法
4. 改良のアドバイス
を教えてください。
```

#### 設計意図

- **観察項目をガイド**: 土壌の特徴を言語化しやすくする
- **pH傾向**: 酸性/アルカリ性の傾向を知ることで対策できる
- **農法との関連**: 自然農法を志向するGrowユーザーに適した情報

#### 改善検討事項

- [ ] 観察項目の選択肢を提示すべきか？（例：色は「黒/茶/赤/灰」から選ぶ）
- [ ] 写真との連携は可能か？（マルチモーダルAI対応）
- [ ] 土壌改良アドバイスをより具体的に

## 改善案

### 案1: 土壌分類を選択式にする

```
{location}で家庭菜園をしています。

この地域の土壌は以下のどれに最も近いですか？
- 黒ボク土（Andosols）：火山灰由来、黒褐色
- 褐色森林土（Cambisols）：山林の土、茶色
- 沖積土（Fluvisols）：川沿いの平地
- グライ土（Gleysols）：水田地帯
- その他

選んだ土壌について、特徴と栽培のアドバイスを教えてください。
```

**メリット**: 回答が安定する、選択肢がGrowアプリと一致
**デメリット**: AIの推論能力を活かせない

### 案2: 段階的な質問

```
{location}の土壌について、3つの質問に答えてください。

Q1. この地域の地質学的背景は？（火山灰、沖積層、丘陵地など）
Q2. 一般的な土壌タイプは？（WRB分類名と日本語名）
Q3. この土壌で野菜を育てる際の注意点は？
```

**メリット**: 簡潔、要点を絞れる
**デメリット**: 詳細情報が不足する可能性
