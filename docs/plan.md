# AISeed コンセプトドキュメント

## 哲学

AISeedは、ニューロダイバージェントの人々が自分の強みを発見し、それを活かして自立することを支援するプラットフォーム。

診断やラベル付けではなく、体験を通じた気づきを大切にする。

AIによりデスクワークのコストが激減していることを活用

---

## スキルズ成長モデル

AIseedの核心は「AIを賢くする」ではなく「ユーザーのスキルズが成長する」こと。

### 従来のAIサービス vs AIseed

| 観点 | 従来のAIサービス | AIseed |
|------|------------------|--------|
| 成長するもの | AI | ユーザー |
| 価値の所在 | サービス側 | ユーザー側 |
| 依存関係 | ユーザーがサービスに依存 | ユーザーが自立 |

### スキルズファイルの成長

```
初期状態:
  スキルズファイル = 空
      ↓
Spark体験:
  「細部に気づく力がある」→ スキルズに追加
      ↓
Growでの実践:
  「毎日観察できた」→ スキルズが具体化
  「変化に気づいた」→ 強みの確認
      ↓
Createでの創作:
  「ビジュアルで表現できた」→ 新しいスキル発見
      ↓
スキルズファイルが育っていく
```

### スキルズファイルの内容

- 発見した強み
- 実践した経験
- 創ったもの
- 成長の記録

### ポータビリティ

```
スキルズファイル（Markdown）
    ↓
どのAIサービスでも使える
    ↓
Claude、ChatGPT、Gemini...
    ↓
ベンダーロックインなし
```

ユーザーは自分のスキルズファイルを持ち出して、他のAIサービスでも活用できる。AIseedに縛られない。

### 成長の価値

```
スキルズファイルが育つ
    ↓
自分の成長が可視化される
    ↓
自己肯定感につながる
    ↓
さらなる成長へ
```

### データ保存ポリシー

サーバーに長期保存するのはスキルズファイルのみ。

| 観点 | 効果 |
|------|------|
| 透明性 | 「あなたのスキルズだけ預かっています」と明言できる |
| 軽量化 | ストレージ・バックアップコスト削減 |
| プライバシー | 個人情報リスク最小化 |
| 規制対応 | GDPR等への対応が容易 |
| 信頼 | ユーザーが安心して使える |

```
Spark/Grow/Createの活動データ:
  → 処理後、スキルズとして抽出
  → 元データは保持しない（またはローカル）

サーバーに残るもの:
  → スキルズファイルのみ
```

### スキルズファイルの管理

**本人による管理**
- いつでも確認可能
- いつでも修正可能
- 完全な透明性

**テスト用データ**
- 本番のスキルズファイルとは別管理
- シナリオ改善・開発に使用
- 個人を特定できない形式

## サービス構成

```
        Spark（入口・気づき）
       ↗︎       ↖︎
      ↑         ↑
   Grow ←────→ Create
```

| サービス | コンセプト | 内容 |
|----------|-----------|------|
| Spark ✨ | 自分を知る | 対話や体験から強みと「らしさ」を発見 |
| Grow 🌱 | 育てる | 栽培・遊び |
| Create 🎨 | BYOA - あなたのAIで創る | Webサイト作成・AIスキルファイル |

3つのサービスは相互に作用し、循環することで「人間としての成長」につながる。

## Spark 体験タスク（子供・親子向け）

### 設計原則

| 原則 | 実装 |
|------|------|
| 質問しない | 体験させて反応を見る |
| 正解がない | 選択に優劣をつけない |
| ラベルを貼らない | 傾向を詩的・やわらかく表現 |
| 押し付けない | 「かもしれません」で提示 |
| 成長する | Grow/Createの経験で更新される |

### タスク例

**タスク1: 観察 - 何が気になる？**
- 森の写真を表示
- 最初に目がいった場所をタップ
- *見ているもの: 全体把握型 vs 細部注目型*

**タスク2: 音と連想**
- 雨音を5秒再生
- 何を思い浮かべたか、イラストから選ぶ
- *見ているもの: 具象的連想 vs 抽象的連想*

**タスク3: 並べてみる**
- バラバラの図形を好きなように配置
- *見ているもの: 規則性志向 vs 自由配置志向*

**タスク4: 続きを選ぶ**
- 短い物語の冒頭を提示
- 続きを3つの絵から選ぶ
- *見ているもの: 環境への志向性*

**タスク5: リズム**
- 画面を好きなようにタップ（10秒間）
- *見ているもの: 規則的 vs 即興的*

### フィードバック

断定せず、可能性として提示：
- 「細かいところに気づく目を持っていますね」
- 「まず動いてみるタイプかもしれません」
- 「自分のリズムを大切にしているようです」

### Grow/Createへの誘導

傾向に基づいて提案：
- 細部注目型 → 「芽の成長を観察してみる？」（Grow）
- 即興型 → 「AIと気ままにおしゃべり」（Create）
- 規則性志向 → 「毎日の記録をつけてみる」（Grow）

## 開発体制

### リソース

| 項目 | 内容                 |
|------|--------------------|
| 開発者 | 自分（1人）             |
| AI開発パートナー | Claude Max（$200/月） |
| 月間運営費 | 既存AdSenseでカバー      |

### Claude Maxの優位性

```
Claude Max $200/月
  → 24時間対応
  → 待ち時間なし
  → ドメイン知識を共有済み
  → 人に依頼するより安価で効率的
```
Claude Max $200/月は、雇用や業務委託する場合では、最低賃金の時給1000円少々で換算すると月30時間分ぐらい。
それよりは、遥かに多くの作業をさせられるはず。
プログラマーやテスターとして、どれぐらい働くかを測定

### AIテスター：シナリオ進化モデル

従来のテスターは「バグを見つける」だけ。AIseedでは、AIをシナリオの共同開発者として活用する。

```
従来のテスター:
  バグを見つける → 報告 → 修正

AIテスター:
  シナリオを試す → 改善案を出す → 進化させる
```

#### 具体的な活用方法

```
Claude Max に様々なペルソナでテストさせる:

「視覚優位の10歳の子供として体験タスクをやってみて」
「ADHD傾向のある大人としてやってみて」
「細部注目型の人がどう反応するか予測して」
    ↓
シナリオの穴が見える
    ↓
フィードバック文言の改善
    ↓
新しいタスクのアイデア
```

#### AI時代の開発サイクル

```
Claude Max でシナリオ検証・改善案生成
    ↓
調整・ブラッシュアップ
    ↓
実際のテスター（人間）で検証
    ↓
さらに改善
    ↓
（繰り返し）
```

AIで荒削りして、人間で仕上げる。

## AI時代の協働モデル

AIseedは「AIによる効率化」ではない。AIと人間が協働して価値を生み出すモデルを示している。

### 従来のAI活用 vs AIseed

| 観点 | 従来のAI活用 | AIseed |
|------|-------------|--------|
| 目的 | 効率化・コスト削減 | エンパワーメント・自立支援 |
| 人間の役割 | AIに代替される | AIと協働して能力拡張 |
| 価値 | 速く・安く | できなかったことができる |

### AISeedが示すAI時代の姿

```
1. 開発: AIがシナリオを進化させる（AIテスター）
2. 提供: AIが読み書きの壁を取り除く（BYOA）
3. 支援: 当事者がAIを使って支援者になる（代理店）
4. 循環: 支援される側 → 支援する側へ
```

AIは人間を代替するのではなく、人間の可能性を広げるもの。

- AIの助けで自分の強みを発見し
- AIの助けでWebサイトを作り
- AIの助けで他の人を支援する仕事を持つ

これがAI時代の一つの理想形。
