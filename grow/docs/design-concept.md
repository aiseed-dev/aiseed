# Grow デザインコンセプト

## 1. デザイン哲学

### 1.1 コアコンセプト：「観る」を助けるデザイン

```
自然農法の本質：
  「何をしたか」ではなく「何が起きているか」を観察する

デザインの役割：
  観察を邪魔せず、記録を助け、気づきを促す
```

### 1.2 デザイン原則

| 原則 | 説明 | 具体例 |
|------|------|--------|
| **静けさ** | 自然の中での観察を邪魔しない | 控えめな色、最小限のアニメーション |
| **透明性** | 土・植物・空が主役、UIは背景 | 写真を大きく、UIは半透明 |
| **手触り** | 土に触れる感覚をデジタルで表現 | 柔らかな角丸、自然な影 |
| **余白** | 呼吸できる空間、詰め込まない | 十分なパディング、行間 |
| **時の流れ** | 成長と変化を感じられる | タイムライン表示、季節感 |

## 7. アニメーション・インタラクション

### 7.1 原則

```
- 控えめ：自然の観察を邪魔しない
- 意味がある：状態変化を伝える
- 高速：200-300ms以内
```

### 7.2 具体例

| 操作 | アニメーション | 時間 |
|------|---------------|------|
| カード表示 | フェードイン + 上スライド | 200ms |
| ボタンタップ | リップル + スケール | 150ms |
| 画面遷移 | スライド（iOS） / フェード（Android） | 300ms |
| 写真拡大 | ヒーローアニメーション | 300ms |
| 成功通知 | 上からスライドイン | 250ms |
| プルリフレッシュ | 植物が伸びるアニメーション | カスタム |

## 9. アクセシビリティ

### 9.1 必須対応

```
- コントラスト比: 4.5:1 以上（本文）、3:1 以上（大文字）
- タップターゲット: 48dp × 48dp 以上
- フォントサイズ: システム設定に追従
- スクリーンリーダー: 全要素にラベル
- カラーブラインド: 色だけで情報を伝えない
```

### 9.2 考慮事項

```
- 畑での使用：明るい屋外でも見やすい高コントラスト
- 汚れた手：大きなタップ領域
- 高齢者：拡大可能なUI
```

## 10. ブランドアイデンティティ

### 10.1 ロゴコンセプト

```
      🌱
     Grow

シンプルな芽のアイコン + Growの文字
- 成長を象徴
- 自然の力を表現
- 最小限の要素
```

### 10.2 トーン＆マナー

```
- 温かい：冷たいテクノロジー感を出さない
- 謙虚：自然に対する敬意
- 実用的：飾りすぎない
- 信頼：データを大切にする姿勢
```

### 10.3 コピーライティング

```
良い例：
  「今日も観察を楽しみましょう」
  「何が起きていますか？」
  「記録する」

避ける例：
  「今すぐ記録！」（押し付けがましい）
  「データを入力」（事務的）
  「登録完了しました！」（システム的）
```
